Lab Projects

Fortinet · Hub-and-Spoke · BGP · IKEv2 · VTI

Lab — ADVPN (Auto-Discovery VPN)

This lab demonstrates a high-performance Hub-and-Spoke VPN architecture using Fortinet's ADVPN technology. The design provides a scalable solution for multi-site organizations by allowing remote branches (Spokes) to automatically discover and establish direct "shortcut" tunnels to one another, bypassing the central Hub for data traffic.

Technical Specifications

  • Infrastructure: Deployed a central Hub connected to multiple remote sites (SPOKE-A and SPOKE-B).
  • Dynamic Shortcuts: Configured Auto-Discovery protocols to allow Spokes to initiate direct dynamic tunnels, reducing latency and Hub CPU utilization.
  • Multi-VLAN Segmentation: Managed traffic across geographically separated subnets — VLAN 10 (10.10.10.0/24), VLAN 20 (20.20.20.0/24), and VLAN 30 (30.30.30.0/24).
  • Routing & Control Plane: Utilized the Hub as a route reflector to manage prefix advertisements and ensure full reachability across the entire overlay network.
  • VPN Tunneling: Standardized on IKEv2 for secure tunnel negotiation, utilizing Virtual Tunnel Interfaces (VTI) for simplified routing management.

Validation

  • Confirmed direct Spoke-to-Spoke communication by monitoring the creation of dynamic "shortcut" entries in the IPsec tunnel list.
  • Verified end-to-end reachability between VPCs in different VLANs across the dynamic fabric.

FortiGate · Cisco ISR · IKEv2 · Route-Based VTI · PSK

Lab — Site-to-Site VPN IPSec (FortiGate ↔ Cisco ISR)

This lab demonstrates a site-to-site VPN tunnel between a FortiGate Firewall and a Cisco ISR Router. The implementation uses a Route-Based approach to enable communication between multiple internal VLAN subnets.

Technical Specifications

  • Tunnel Architecture: Configured a Virtual Tunnel Interface (VTI) on the Cisco side (interface Tunnel1) and a phase1-interface on the FortiGate.
  • IKE Version: Utilized IKEv2 for the key exchange protocol.
  • Security Suite: Implemented DES encryption with SHA-256 integrity and Diffie-Hellman Group 14.
  • Authentication: Secured the connection using a Pre-Shared Key (PSK).
  • Inter-VLAN Connectivity: Enabled traffic flow between local subnets by routing traffic through the dedicated tunnel interfaces.

Validation & Monitoring

  • Tunnel Status: Verified the connection using show crypto ikev2 sa to ensure the status is READY.
  • Traffic Flow: Confirmed data transmission by monitoring the increase in Encapsulation and Decapsulation packets.
